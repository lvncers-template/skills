# Slide Draft

## 概要

箇条書きメモ・キーワードをもとに、社内共有・LT用の発表原稿を一緒に作るコマンド。

## 使い方

```sh
/slide-draft [箇条書きメモ・キーワード・ファイルパス]
```

## AIの動き（ステップ）

### STEP 1 — 発表時間だけ確認（1問のみ）

社内LT想定なので聴衆の説明は不要。
時間だけ確認して即スタート。

```md
発表時間を教えてください。
A) 5分以内（LT）
B) 10〜15分
C) 30分〜
```

| 時間     | ページ数目安 |
| -------- | ------------ |
| 5分以内  | 5〜8ページ   |
| 10〜15分 | 10〜15ページ |
| 30分〜   | 20ページ〜   |

### STEP 2 — メモの整理

- 事実確認・誤字脱字の修正
- 言いたいことの抽出と整理
- 情報の不足・矛盾があれば指摘・補完候補を提示

### STEP 3 — 構成の提案（確認あり）

```md
## 構成案（LT 5分 / 7ページ想定）

| #   | セクション | 主な内容      |
| --- | ---------- | ------------- |
| 1   | タイトル   | タイトル      |
| 2   | 背景・課題 | 背景の説明    |
| 3   | やったこと | 概要          |
| 4   | 結果       | 成果          |
| 5   | まとめ     | 3つのポイント |

承認しますか？追加・変更・削除があればどうぞ。
```

### STEP 4 — 原稿執筆

承認後、Markdownで原稿を書く。

### STEP 5 — レビューと確定

- 人間が確認・修正を返す
- 反映して再提案
- 「OK」で原稿確定 → `/slide-slidev ./draft.md` で次のフェーズへ
