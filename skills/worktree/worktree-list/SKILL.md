# Git Worktree 一覧

## 概要

worktree の一覧と状態を取得し、報告用テーブルで出力する。

## 手順

### 1. worktree 一覧の取得

```sh
git worktree list
```

### 2. 各 worktree の状態収集

各 worktree ごとに次を確認する。

- **未コミット変更**: 対象 worktree のディレクトリで `git status` を実行
- **lock 状態**: 一覧に `locked` と出ているかで判定
- **ディレクトリ存在**: 当該パスが実在するか

```sh
# 対象 worktree のディレクトリで
cd <worktree-path> && git status --short
```

### 3. 報告フォーマット

次のテーブル形式で一覧を報告する。

```
┌──────────────────────┬─────────────────┬────────────────┬──────┬──────────────┬────────────────┬──────────────┐
│ worktree             │ ブランチ        │ 未コミット変更 │ lock │ ディレクトリ │ 削除可否       │ 備考         │
├──────────────────────┼─────────────────┼────────────────┼──────┼──────────────┼────────────────┼──────────────┤
│ <ディレクトリ名>     │ <ブランチ名>    │ <件数 or ->    │ -/🔒 │ 存在/なし    │ ✅/⚠️/❌       │              │
└──────────────────────┴─────────────────┴────────────────┴──────┴──────────────┴────────────────┴──────────────┘
```

**削除可否の目安:**

- **✅ 削除可**: 未コミット変更なし・lock なし・ディレクトリ存在
- **⚠️ prune のみ**: ディレクトリがすでになく参照だけ残っている（`git worktree prune` のみ提案）
- **❌ 要対応**: 未コミット変更あり、または lock あり
